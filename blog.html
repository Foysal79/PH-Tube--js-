Discuss the scope of var, let, and const .

answer :

var -> var হল একটি কীওয়ার্ড যা জাভাস্ক্রিপ্টে ভেরিয়েবল ঘোষণা করতে ব্যবহৃত হয় যা ফাংশন স্কোপড। ES6 আবিষ্কার আগে জাভাস্ক্রিপ্টের সমস্ত কীওয়ার্ড শুধুমাত্র “var” কীওয়ার্ড দিয়ে ঘোষণা করা হয়েছিল। var কীওয়ার্ডটি গ্লোবাল-স্কোপ ভেরিয়েবল ঘোষণা করতেও ব্যবহৃত হয়।


let let -> JavaScript "let" একটি কীওয়ার্ড যা জাভাস্ক্রিপ্টে ভেরিয়েবল ঘোষণা করতে ব্যবহৃত হয় যা ব্লক স্কোপড। জাভাস্ক্রিপ্টের 'ES6' বা 'ES2015' সংস্করণে দুটি নতুন কীওয়ার্ড যোগ করা হয়েছে। সাধারণত, জাভাস্ক্রিপ্টের সাথে কাজ করার সময় আমাদের অবশ্যই "let" কীওয়ার্ড ব্যবহার করার পরামর্শ দেওয়া হয়।


const const -> ES2015 (ES6) একটি নতুন ভেরিয়েবল সংজ্ঞায়িত করার জন্য const কীওয়ার্ড চালু করেছে। অন্যদের কাছে কনস্ট পরিবর্তনশীল ঘোষণার পার্থক্য হল যে এটি "Reassigned" করা যাবে না।

What do you mean by REST API?

answer :

REST API -> রিপ্রেজেন্টেশনাল স্টেট ট্রান্সফার (REST) ​​হল একটি স্থাপত্য শৈলী যা ওয়েব পরিষেবা তৈরির জন্য ব্যবহার করা সীমাবদ্ধতার একটি সেট সংজ্ঞায়িত করে। REST API হল কোনো প্রক্রিয়াকরণ ছাড়াই সহজ এবং নমনীয় উপায়ে ওয়েব পরিষেবাগুলি অ্যাক্সেস করার একটি উপায়৷ .

REST প্রযুক্তি সাধারণত আরও শক্তিশালী সিম্পল অবজেক্ট অ্যাকসেস প্রোটোকল (SOAP) প্রযুক্তির চেয়ে পছন্দ করে কারণ REST কম ব্যান্ডউইথ ব্যবহার করে, সহজ এবং নমনীয় করে এটি ইন্টারনেট ব্যবহারের জন্য আরও উপযুক্ত করে তোলে। এটি একটি ওয়েব পরিষেবা থেকে কিছু তথ্য আনয়ন বা দিতে ব্যবহৃত হয়। REST API এর মাধ্যমে করা সমস্ত যোগাযোগ শুধুমাত্র HTTP অনুরোধ ব্যবহার করে।
